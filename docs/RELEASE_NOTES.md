# 发布说明

这是 `qqbot-framework` 当前整理后的可交付发布版说明。

## 最新更新（AI 中转站 / 命令路由修复）

本次更新重点：

- 新增 **AI 中转站模型列表获取** 能力
  - 新命令：`AI模型列表`
  - 通过 OpenAI 兼容接口 `GET /v1/models` 拉取可用模型
  - 自动提取 `data[].id` 作为模型名
- 优化 **AI 中转站配置流程**
  - `配置AI中转站` 现在只需要：`地址 Key`
  - 不再要求用户配置时先手填模型名
  - 配置成功后会提示下一步执行 `AI模型列表`
- 新增 **按序号选择模型**
  - 新命令：`选择AI模型 序号`
  - 获取模型列表后可直接按编号切换，避免手打完整模型名
- 保留 **按模型名切换** 能力
  - 继续支持：`切换AI模型 模型名`
- 修复 **unknown_command_hint 误伤 URL 命令** 的问题
  - 之前消息中只要包含 `/`，例如 `https://.../v1`，就可能被“未知命令提示”插件错误拦截
  - 现已调整为只在更明确的未知命令场景下触发，不再影响 `配置AI中转站 https://.../v1 sk-...`
- 更新帮助文案
  - `AI帮助`
  - 总帮助 `help`
  - 现在都会展示新的 AI 模型列表 / 选择模型命令

当前 AI 中转站相关命令为：

- `AI帮助`
- `AI状态`
- `AI 你的问题`
- `问AI 你的问题`
- `配置AI中转站 地址 Key`
- `AI模型列表`
- `选择AI模型 序号`
- `切换AI模型 模型名`

## 最新更新（图片卡片长图优先 / 超限再分页 / 修复底部裁切）

本次更新重点优化了机器人图片卡片的阅读体验，尤其是长文本回复：

- 优化 **图片卡片分页策略**
  - 不再简单按固定条数分页
  - 改为优先根据内容自动生成单张长图
  - 只有在高度超过阈值时，才自动分页
- 新增 **按实际内容高度分页**
  - 不同长度的文本会按真实排版高度判断是否拆页
  - 比固定 10 条一页更自然，也更适合菜单、帮助、说明类内容
- 修复 **长图底部内容被裁切** 的问题
  - 之前图片卡片在估算总高度时，与实际绘制时使用的行高不一致
  - 会导致图片虽然生成成功，但底部文字被画到画布外面，看起来像“长图不全”
  - 现已统一高度估算与实际绘制逻辑，确保内容完整显示
- 当前图片卡片行为调整为：
  - **优先长图**
  - **内容完整显示**
  - **放不下再分页**

这次调整主要影响文件：

- `app/card_mode.py`
- `app/renderers/card_image.py`

## 最新更新（菜单系统 / 插件列表增强 / 模糊提示）

本次继续完善了机器人交互体验：

- 新增 **总菜单系统**
  - 新命令：`菜单`
  - 用户发送 `菜单` 后，可看到当前机器人所有主要功能入口
- 新增 **分类子菜单**
  - `通用菜单`
  - `签到菜单`
  - `AI菜单`
  - `插件菜单`
  - `群管菜单`
  - `卡片菜单`
  - `语录菜单`
  - `龙虾菜单`
  - 每个子菜单都会展示对应功能说明、命令列表和使用示例
- 优化 **总菜单版式**
  - 将原本较生硬的帮助文本改为更适合 QQ 阅读的菜单样式
  - 菜单入口更集中、更清晰
- 增强 **插件列表可读性**
  - `插件列表` 现在会显示：
    - 插件名
    - 启用状态
    - 来源（系统内置 / 扩展插件）
    - 类型（命令 / 关键词 / 正则）
    - 描述
    - 版本 / 作者 / 触发方式
- 新增 **插件详情命令**
  - 新命令：`插件详情 插件名`
  - 可查看某个插件的完整信息
- 增强 **未知命令提示 / 模糊匹配**
  - 用户输入不准确命令时，会自动提示最接近的可用命令
  - 如果不清楚功能，也会引导用户直接发送 `菜单`
- 修复 **AI 对话被未知命令提示插件误拦截** 的问题
  - 之前像 `AI 你好` 这样的已知 AI 命令，可能被 `unknown_command_hint` 抢先处理，导致看起来“AI 不回复”
  - 现已修复：只有在真正需要提示未知命令时，提示插件才会接管消息
- 新增 **更新检测插件**
  - 新命令：`检查更新`
  - 新命令：`更新状态`
  - 支持手动检查当前机器人是否落后于 GitHub 最新版本
- 新增 **自动提醒新版本**
  - 机器人后台会定时检查 GitHub
  - 如果发现新版本，会私聊提醒主人
  - 同一远端版本只提醒一次，避免刷屏
- 更新检测改为 **双模式兼容**
  - Git 工作区部署：直接比较本地 git commit 和 GitHub 最新 commit
  - 非 git 部署：改为读取本地 `BUILD_INFO.json` 再和 GitHub 比较
  - 两种部署方式都能支持更新检测/自动提醒
- 新增 **Git 一键部署脚本**
  - 新文件：`deploy/bootstrap-git-install.sh`
  - 适合需要保留 `.git`、后续使用 `git pull`、以及完整更新检测的人
  - 该脚本会直接 `git clone` 仓库并进入现有安装流程
- 新增 **正式版本号文件**
  - 新文件：`VERSION`
  - 当前版本号可被打包、安装、更新检测、自动更新流程共同使用
- 更新检测现在支持 **显示版本号**
  - 提示里不再只显示短 commit
  - 会优先显示：`版本号（短提交号）`
  - 例如：`v0.3.0（3f9ac50）`
- 自动更新结果通知也支持 **显示更新前 / 更新后版本号**
  - 更容易看出当前机器人具体升级到了哪个版本
- 更新内容提示支持 **中文人话摘要**
  - 不再直接原样展示英文 commit message
  - 会尽量按“新增 / 修复 / 优化 / 自动修复”等中文方式说明本次改了什么
  - 会尽量提示更新的是功能、插件、安装器还是 bug 修复
- 新增 **全局卡片模式切换**
  - 新命令：`卡片模式`
  - 新命令：`切换卡片模式 文字`
  - 新命令：`切换卡片模式 图片`
  - 切换为图片模式后，机器人后续大部分文字回复会自动转成图片卡片
- 新增 **图片卡片基础能力**
  - 新增 Pillow 图片卡渲染器
  - 新增静态图片访问路径 `/static/...`
  - 新增测试命令：`测试图片卡`
- 修复 **图片卡中文乱码问题**
  - 安装并使用中文字体，解决图片卡中文显示异常
- 修复 **图片卡内容显示不全问题**
  - 增加自动换行
  - 支持长内容自动分页
  - 菜单等长文本在图片模式下会拆成多张卡片发送

## 1. 当前推荐方案

- 业务框架：`qqbot-framework`
- QQ 适配器：`NapCat`
- 协议：`OneBot v11`
- 消息 API：HTTP
- 事件上报：HTTP POST

## 2. 这份发布包包含什么

发布压缩包名称：

```text
qqbot-framework-release.tar.gz
```

打包内容包含：

- `app/` 主程序
- `deploy/` 部署模板
- `docs/` 文档
- `scripts/` 打包脚本
- `user_plugins/` 示例和自定义插件
- `.env.example`
- `Dockerfile`
- `docker-compose.yml`
- `install_plugin.py`
- `requirements.txt`
- `run.sh`
- `README.md`
- `DEPLOY_FINAL.md`

## 3. 这份发布包默认不包含什么

为了避免泄露敏感信息与运行时状态，发布包应尽量排除：

- `.env`
- `.venv/`
- `logs/`
- `data/`
- `ntqq/`
- `napcat/cache/`
- 本机二维码缓存
- 当前服务器登录态

## 4. 当前已具备的主要能力

- OneBot v11 事件接收
- HTTP API 回消息
- 插件自动发现与自动加载
- 插件启用 / 禁用 / 升级 / 卸载
- 插件市场索引
- 主人命令控制
- 签到 / 积分系统
- 高级群管插件 v2
- 无斜杠命令兼容（如 `ping` 和 `/ping` 都可用）
- NapCat 接入 QQ 扫码登录
- SQLite 持久化

## 5. 当前已知限制

目前框架主要稳定支持：

- 文本消息
- 群管理 OneBot API
- 命令 / 关键词 / 正则插件

尚未做统一高级封装：

- JSON 卡片消息
- XML 卡片消息
- 图片 / 文件 / 语音等通用消息段构造器
- 更完整的 notice / request 事件体系

## 6. 新服务器部署时需要关注

至少配置：

```env
ONEBOT_API_BASE=http://127.0.0.1:3000
QQBOT_OWNER_IDS=你的QQ号
```

建议优先阅读：

- `README.md`
- `docs/DEPLOY_GUIDE.md`
- `docs/PLUGIN_GUIDE.md`
- `deploy/NAPCAT_DEPLOY.md`
- `DEPLOY_FINAL.md`

## 7. 使用建议

如果你要把这套发给别人部署：

1. 发 `qqbot-framework-release.tar.gz`
2. 提醒对方自己填写 `.env`
3. 提醒对方自行扫码登录 NapCat
4. 不要携带你自己的登录态和数据库

## 8. 重新打包命令

```bash
./scripts/package_release.sh
```

执行成功后会生成：

```text
qqbot-framework-release.tar.gz
```
